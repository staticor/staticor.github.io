同一用户连续窗口计算

计算留存~连续登录问题

输入用户id, 日期date.
输出:求有过连续活跃的用户占比.


uid, dt

1002 2021-12-12

需求， 求连续登录5天以上的用户占比。

思路一， join5次

select  。。。。 
from t   as ta
join t as tb on   ta.uid = tb.uid and date_add(ta.dt, 1) = db.dt 
。。。 连续写多次join

很明显，这不是一个优雅的解法，会被骂。

思路二，窗口函数 row_number

  1. 将原始数据去重
  2. 同一用户按dt排序， 再进行一个辅助列 rank ，用row_number排序
  3. 计算 dt和 rank的差值 delta， 这个差 就表示的连续登录的起点 计算 <uid, delta> 的count数

简单代码如下：

 select uid, date_sub(dt, rank) as delta 
 from (select uid, dt, row_number() over(partition by uid order by dt ) rank 
 from (select uid, dt  from T group by uid, dt ) ta
 ) tb 
 group by uid, date_sub(dt, rank)
 having count(1) >= 5

思路三， 使用LAG（LEAD） 函数

这一对函数是专门用来在窗口中计算和上x行的差值的， 在查询语义上我认为是更易联想到的解法（较思路2）


  ( select uid, dt
       ,  lag(dt, 4, '0000-00-00') over (partition by  uid order by dt) lag_dt1  
        ,  lag(dt, 3, '0000-00-00') over (partition by  uid order by dt) lag_dt2
        ,  lag(dt, 2, '0000-00-00') over (partition by  uid order by dt) lag_dt3
        ,  lag(dt, 1, '0000-00-00') over (partition by  uid order by dt) lag_dt4
    from T )  tmp 
....

如果当前日期和前面窗口的几个元素作差， 比较出既出现了“差一天我也来了” “差两天我也来了“ ... ...
那么说明这一天就是我的第五天登录。
