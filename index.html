之前我用不同的职业title来去考虑不同数据行业的工作特性， 今天我想换一种视角。
考虑处理问题和平时的交流对象来看待自己的工作。

在考虑做什么之前， 要思考一个问题： 数据从哪里来， 数据要向哪里去。

  • pipe（管道流）自己是属于数据flow的哪一环；
  • hot-code, use （自己生产的数据是直接用于客户吗？ 是用于公司内部？）

因为我个人更熟悉的数据行业是距离业务数据（企业内生产的产品、用户数据）， 所以我按照数据规模分成了下面的三种。

小数据、商业分析

数据分析师的一种， 侧重于企业的宏观数据。 例如每月产品的成交、用户激活、注册数量。 向管理层或外部三方audit提交自己的核心数据。
几乎不会涉及到单个客户的明细数据。

这类方向一般是要求具有3年+数据分析的工作经验， 擅长BI报告的数据解读， 对公司业务模式有深刻的理解， 并且对各类的基础归因分析信手拈来（自然趋势、季节效应、活动波动）。

对编程要求并不高， 纯SQL操作就能满足大部分数据提取的任务， 复杂数据可以由上游处理（数据仓库、数据挖掘）

虽然处理的数据平常只用Excel就能解决， 不过因为影响的面和汇报层级较高， 所以在企业中有较高的话语权， 可以视作企业内参谋的决策。而且也由于汇报的需要，只需要重点数据而非全面数据。要求人的能力是归因， 即能对增长、停滞给出解释。

特点：数据一般是由专业工程师准备好， 商业分析与战略分析像大厨一样需要将配好的食材加工成可口的数据菜肴， 交付到高层。

成长路径： 初级数据分析师， 商业BI分析师

初级数据分析

数据分析对我而言是一个浅者觉其浅， 深者觉其深得无法自拔的四个字。
浅是指很多刚毕业的学生可以选择数据分析为初期准备工作，一边工作，一边观望，然后2-3年之后找到自己适合的分支方向去发展。

我虽然从事过一年多的数据分析工作，但不愿意和人谈起数据分析和数据挖掘的区别。
原因是有的不懂”数据分析“的深处的人， 总把这个工作想得容易。
因为在很多时候分析数据需要挖掘的技能， 而挖掘出来的结果还要分析报告的交付。 二者在我看来是一个方向， 也许是有的地方一直以来让两个人做不同的侧生工作，才有了数据分析师和数据挖掘两类职业。 但我则一直相信这是对人才放低门槛的妥协作派， 并不认可。

虽然PPT与数据可视化也固然重要， 但也许在学生阶段不应将主要精力放在PPT和美学研究这上。 我也见过一些自称为商业分析师的人， 他们可能更擅长的是PPT制作，而非分析。 先学会用最简单的图表讲述故事， 简单生动， 直接深刻。 比起花哨而不知所云， 有用的多。

有小型企业初期是没有足够数据的， 所以有的企业要借助外部“在一定权限内可以”拿到的数据， 比如我们现在有很多企业信息查询的服务， 他们是在工商网拿到企业法人、注册信息。 这就需要专业的爬虫工程师， 基本上要求Python开发和熟悉当下流行的反爬策略。

那么下面介绍什么是KKT条件。

KKT条件(Karush-Kuhn-Tucker Condition)

1） 满足 L 函数的导数为0；
2) 满足 所有不等约束取等号的条件；
3) 满足 $\lambda_i g_i(x) = 0 $, 又称KKT松弛互补条件。

在支持向量机中， 利用第三个条件能快速得到：

  • 如果是不在分类边界上的点， 它们一定满足 $ h_i(x) < 0 $ 则对应的 $\lambda_i$ 系数为0；
  • 如果在分类边界上的点， 它们一定满足 $ h_i(x) = 0 $ 则对应的 $\lambda_i$ 系数为正数； 这些点就称为 支持向量

Soft-Margin SVM

软间隔SVM是指对训练集样本均满足线性可分这一约束条件进行放松， 允许有一些样本是误分类的。
回顾之前提到的线性均可分是指所有样本的函数间隔都大于等于1:

那么，如果说有些异常样本(outliers)不满足这个条件， 它们属于误分类（说明它们的函数间隔小于1）， 我们就令超出部分的间隔距离为参数 $\xi_i （\xi \ge 0, i = 1,2 … N ）$

Kernel Method

前面硬间隔与软间隔都针对的是线性空间分类问题， 是不能直接处理非线性场景的。
因此有人提出了将低维空间转换为高维空间，在高维空间里再借助软硬间隔分类器。

核方法（核函数）就是遵循了这种思路， 找到某个映射函数： $ \Phi: \mathcal{X} \rightarrow \mathscr{Z}$ 使得原来空间的内积运算能够映射到高维空间中。 即，能找到这样一个函数 $\mathscr{K}(x_i, x_j) = <\Phi(x_i), \Phi(x_j)> $
我们称这个函数K为核函数。
通过观察发现在原来的软硬间隔优化问题当中， 其实对原始样本的计算只涉及到内积计算。因此我们没有必要去找到这个具体的映射， 而是 直接去用这个核函数的计算结果而非去寻找中间的映射函数$\Phi$，这种技巧称为核技巧（Kernel Trick） 。

正定核： 对原样本组成的Gram矩阵如果是半正定的， 就称这样的核为正定核。

绪论与学习资料

2019年9月发现了B站上一个出色机器学习资料， 搜索《白板机器学习》 UP主老师的深刻和广度给我留下了印象。

两大视角

  • 频率派视角
  • 贝叶斯派视角

频率派是认为生成观测数据（Observed Data）的参数是常量，因此使用最大似然方法去求解这个参数。
或者是引入 loss function， 将问题转变为优化问题；
使用梯度下降或者牛顿方法去求解这个参数。

有代表性的方法是线性回归、支持向量机。

贝叶斯派是认为未知参数 $\theta$ 也是服从某未知分布的随机变量（random variable),
所以对其计算就需要求大最大后验了。 而后验则需要借助贝叶斯定理，有代表性的大模型就是概率图。
知名的算法有MCMC、条件随机场等。

Perception（感知机）

假设我们拥有若干个观测样本是来自两个分类总体，并且在空间中存在一个超平面，将这些样本像象棋一样，一条“楚河汉界”就将原来的样本分成正负两类， 我们称这个样本满足线性可分数据集条件（Linear Seperable Dataset).

上世纪1957年Rosenblatt提出了Percentron感知机模型，后来人们在此基础上改进，感知机可以算是神经网络与支持向量机的上一代模型。

感知机模型是一个线性分类器，能够处理一些简单的二分类问题。 虽然它并不复杂， 但是通过学习它的模型、策略、算法， 能够让我对机器学习的整体形成一个快速框架感。

是我非常喜欢的一类模型。 大家可以在李航老师的《统计学习方法》一书第二章中找到更详尽的资料。
