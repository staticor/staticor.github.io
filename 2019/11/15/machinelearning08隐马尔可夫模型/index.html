machinelearning08隐马尔可夫模型

Posted on 2019-11-15 | Edited on 2019-10-14 | In 机器学习 | Views:
Symbols count in article: 6.3k | Reading time ≈ 6 mins.

重温李航老师的《统计学习方法》，去年看到隐马的时候到第二小节就放下了，新时光重读老书，别有一番滋味。

隐马尔可夫模型（Hidden Markov Model)是可用来标注问题的统计学习模型，在NLP中有极广泛的应用， 其本质是指我们得到一组观测序列，其背后是由看不到的隐藏变量生成的。 联想到上帝投骰子，我们只看到若干点数，但不知道的是使用的哪个骰子（隐变量是指选择了某个骰子的概率）。

模型定义

隐马尔可夫模型是伴有时间性质的概率模型，时间不一定是等间隔的严格时间序列，但是要有先后时序关系的数据。 描述由一个隐藏的马尔可夫链随机生成不可观测的状态序列（隐变量），由各个状态生成一个观测随机序列。

隐马尔可夫模型的参数由三部分组成$\{A, B, \lambda \}$

  • A 状态转移矩阵（State Transition Matrix)
  • B 发射矩阵，观测矩阵（Emission Matrix)
  • $\pi$ 初始状态向量

这里为了后续公式的统一， 也给出和《统》一书相同的符号约定。
$Q$是所有可能的状态的集合， $V$是所有可能的观测集合，状态集的容量是N， 观测集的容量是M

两段等长的序列，分别是状态序列I和观测序列O:

下面分别对模型参数三部分定义及解释。

状态转移矩阵

A矩阵是一种状态间的描述， 与时刻无关（因为只能是两个相邻的时点）， 同一行各元素相加之和为1.

矩阵中的每个元素表示含义是上一状态是 $i_t：q_i$, 转到下一状态 $i_{t+1} : q_j$的概率是多少。
因为有N个转移方向（包括下一时刻还是自己）， 所有情况的转移概率之和是1.

发射矩阵

发射 二字表示的当前时刻如果状态知道了，可能发出哪些观测值。 （HMM假定了状态和观测值均是离散取值，对于连续情况， 需要考虑其它的模型， 比如卡尔曼滤波等）。

B发射矩阵的尺寸是 N * M, 每行是某个状态， 每列则是由该状态得出某观测值的概率：

模型假设与要解决的问题

对于隐变量模型来说，HMM的假设条件也相对简单， 它有两个假设：

(i)齐次马尔可夫假设， 假设隐藏马尔可夫链在当前时刻的状态只依赖于前一状态， 与历史再之前的状态无关，与当前t的取值无关, 与之前的观测序列无关：

(ii) 观测独立性假设， 任意时刻的观测值只依赖于当前的状态，与历史状态无关，与历史值无关：

记忆tips: 上帝的骰子只取决于当前用的哪颗，和之前的投掷行为无关。

观测序列生成

有了以上2个假设和模型参数， 我们就能用数学语言描述观测序列 {$o_1, o_2, …o_T$}是如何生成出来的了：

输入信息： $ \lambda = (A, B, \pi ) $, 需要的观测序列长度T；
输出序列： $ O = (o_1, o_2, … … o_T)$

  • 初始：令t = 1, 按初始状态向量 $\pi$ 产生状态 $i_t$;
  • 发射：按发射矩阵B， 取值 $b_1(k)$,生成对应的 $o_1$
  • 转移：按状态转移矩阵A，取$ a_{ij} $ 转移到下个状态 $i_{t+1}$
  • 判停：令t = t+1, 如果$ t < T$, 则跳至发射步；
  • 停止：如果 $t+1>T$, 停止。

三个基本问题

那么HMM模型定义后，能解决哪些问题呢？
(1) 最直接的自然是概率计算问题, 给定了模型参数还有观测序列，能否告诉我有多大概率在模型$\lambda$的情况下， 观测序列O出现？ 即求 $P(O|\lambda)$
(2) 学习问题
学习问题就是在不知道参数， 只有观测序列的情况下， 去推测到底数据是怎么来的。 即估计模型的参数 $ \lambda = (A, B, \pi ) $， 使得此时观测序列条件概率 $P(O|\lambda)$ 最大。
能否用极大似然方法直接求出来呢?
(3) 解码问题 (Decoding)
已知模型参数和观测序列，要反推出模型的状态序列。 因为2个序列长度相等， 就像是已知密文求对应明文一样， 称为Decoding问题。

概率计算问题(Evaluation)

首先书里先讲的一种概念上可行但计算不可行的方法，然后再给出有递归性质的前向(forward)与后向(backward)方法。

直接计算法

按前面给出观测序列生成的机制，可以写出来给定模型 $\lambda = (A, B, \pi)$ 观测序列 $O = (o_1, o_2, …., o_T)$ 的 概率 $P(O|\lambda)$, 思路是列举出所有可能的长度为T的状态序列 $I = (i_1, i_2, … i_T) $, 单独计算出在一个状态序列下生成O的概率， 然后求出所有可能, 这里因为序列序列的长度为T， 每个元素都有N种取值可能， 所以直接概率计算方法就相当于是对 $ N^T$ 项进行求和， 在计算量上很大。
